
# Vibe Coding 
*CS 362 – Artificial Intelligence & Machine Learning*

---

## What is Vibe Coding?
**Vibe coding is describing what you want and using AI to help generate and improve code.**  
Instead of writing everything manually, you guide the AI, review its output, and refine the solution.

---

## Core Workflow
1. Describe the task clearly so the AI understands your intent  
2. Generate code using an AI tool  
3. Read and understand what was produced  
4. Test the code with different inputs  
5. Refine your prompt and repeat as needed  

---

## Best Practices

### Be Specific
Vague prompts lead to vague or incorrect code. Clearly state what you want, including features, constraints, and expected behavior.

---

### Start Small
Break problems into smaller steps and improve one thing at a time. This helps you stay in control and makes it easier to debug issues.

---

### Read the Code
Always review the AI-generated code before running it. Understanding the code ensures you can explain it and catch potential mistakes.

---

### Test Everything
Run your program with normal, incorrect, and edge-case inputs. AI-generated code often works for simple cases but can fail in unexpected situations.

---

### Don’t Trust Blindly
AI can produce incorrect logic, inefficient solutions, or even non-existent functions. You are responsible for verifying correctness.

---

### Iterate Your Prompt
If the output is not what you expected, revise your prompt with more detail or constraints. Vibe coding works best as an interactive conversation.

---

### Ask for Explanations
Request explanations of the generated code to deepen your understanding. This turns AI from a shortcut into a learning tool.

---

### Keep It Simple
AI often adds unnecessary complexity or extra features. Ask it to simplify solutions so they match your current skill level.

---

### Use AI as a Tool
AI is useful for generating ideas, starting code, and improving structure. It should support your thinking, not replace it.

---

### Stay in Control
You decide what code to keep, modify, or discard. Treat AI as an assistant, not the final authority.

---

## Helpful Prompt Examples
- “Improve readability and add comments”
- “Add error handling for invalid input”
- “Rewrite using a loop”
- “Simplify this for a beginner”
- “Explain what this program does step by step”

---

## Key Reminder
> **AI writes code fast. You make it correct.**
